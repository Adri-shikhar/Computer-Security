Security Best Practices

Industry standards and recommendations for password security

Essential Security Guidelines

Do's
  • Use Argon2id or PBKDF2-SHA256 for password hashing
  • Generate cryptographically secure random salts (16+ bytes)
  • Implement rate limiting on authentication endpoints
  • Use constant-time comparison for hash verification
  • Add pepper (server-side secret) for defense-in-depth
  • Prevent password reuse with history tracking
  • Enforce strong password requirements (12+ chars, complexity)
  • Regularly review and update hashing parameters
Don'ts
  • Never use MD5 or SHA-1 for password hashing
  • Don't implement your own cryptographic algorithms
  • Avoid storing passwords in plain text or reversible encryption
  • Don't use the same salt for all users
  • Never log or display password hashes
  • Don't use weak iteration counts (PBKDF2 needs 600k+)
  • Avoid string comparison for hash verification (timing attacks)
  • Don't send passwords over unencrypted connections
Recommended References
  • OWASP Password Storage Cheat Sheet
  • NIST Digital Identity Guidelines
  • RFC 9106: Argon2 Memory-Hard Function
  • RFC 8018: PKCS #5 PBKDF2 Specification
Key Takeaways
  1. Use Modern Algorithms: Argon2id is the gold standard, PBKDF2 is acceptable
  2. Always Salt: Unique, random salt per password prevents rainbow table attacks
  3. Add Pepper: Server-side secret provides additional security layer
  4. Constant-Time Operations: Prevents timing attacks during verification
  5. Track Password History: Prevents users from reusing old passwords
