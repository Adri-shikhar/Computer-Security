Why Modern Hashing Matters

MD5/SHA-1: Fast = Vulnerable. Attackers can try billions of passwords per second.

Argon2id: Slow + Memory-Hard = Secure. Deliberately designed to resist GPU/ASIC attacks.

Salting: Two users with "password123" get different hashes, preventing rainbow table attacks.

Algorithm Information
Hash Function Comparison

MD5: Vulnerable
- Instant hashing (~1-5ms)
- Easily cracked with GPUs
- For educational demo only

SHA-1: Weak
- Very fast (~2-8ms)
- Deprecated for passwords
- Historical reference

BCrypt: Secure
- Configurable cost (2^4 to 2^14)
- ~50-200ms per hash
- Production ready

Argon2id: Most Secure
- Memory-hard algorithm
- ~200-1000ms per hash
- Recommended for all new systems

Salt Demonstration - Same Password = Different Hashes

Register multiple users with the exact same password using BCrypt or Argon2. Notice how each hash is completely different due to unique salts.

Why This Matters: Without salts, attackers could use pre-computed rainbow tables. With salts, every password must be cracked individually, making mass breaches nearly impossible.
