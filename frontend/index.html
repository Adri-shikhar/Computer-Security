Automatic Security Upgrade

When a user with MD5 hash logs in:

  1. Verify old MD5 hash
  2. Re-hash password with Argon2id
  3. Update database automatically
  4. Display upgrade notification
Real-World Use: This simulates how companies like Google/Facebook upgrade legacy accounts without forcing password resets.
Security Information
Algorithm Comparison

MD5: Vulnerable - Instant hashing, easily cracked

SHA-1: Weak - Fast hashing, deprecated for passwords

BCrypt: Secure - Adaptive cost factor (10)

Argon2id: Most Secure - Memory-hard, 2 iterations, 64MB memory
